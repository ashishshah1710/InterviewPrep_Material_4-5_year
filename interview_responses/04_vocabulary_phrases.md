## 2. Technical Jargon Explained Simply

- **API (Application Programming Interface):** A set of rules for how programs interact with each other.
- **Bug:** An error or flaw in software.
- **Refactor:** To improve code structure without changing its functionality.
- **Repository:** A storage location for code, often managed by tools like Git.
- **Frontend:** The part of a product users interact with.
- **Backend:** The part of a system users donât see; manages data and logic.
- **CI/CD (Continuous Integration/Continuous Deployment):** Automating code merging and software delivery.
- **Cloud Computing:** Using remote servers for storage and computing instead of local machines.
- **MVC (Model-View-Controller):** A way to organize code by separating logic, UI, and data handling.
- **Tech Debt:** The cost or risk incurred when shortcuts are taken in code.

**Example:**
> *Our backend uses an MVC structure, which separates the data handling from the user interface, making the code easier to manage.*
